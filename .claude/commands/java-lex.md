## タスク

対象ファイル: `$ARGUMENTS`

`$ARGUMENTS` が空の場合は `src/main/java/juml/core/formats/java/JavaLexer.java` を対象にする。

### 手順

1. **対象ファイルを Read で確認する**

2. **JavaLexer の観点で以下を分析する**

   `JavaLexer` は `JavaToken.Type` として次を生成する:
   - `IDENT` — 識別子・キーワード（`isKw()` で区別）
   - `NUMBER` — 数値リテラル
   - `STRING` — 文字列リテラル `"..."`
   - `CHAR` — 文字リテラル `'...'`
   - `PUNCT` — 区切り記号 `( ) { } [ ] ; , . @`
   - `OP` — 演算子（複合演算子 `>>`, `<<=` 等も含む）
   - `EOF` — 終端

   チェックする観点:
   - テキスト入力 / ソースに特殊なリテラル（テキストブロック `"""`, `'\\u'` Unicode エスケープ等）があるか
   - `>>` と `>>>` などの複合演算子の優先順位が正しく解釈されるか
   - ネストしたジェネリクス `List<Map<K,V>>` の `>` が OP として正しくトークン化されるか
   - `@interface` の `@` と `interface` が別トークンになっているか
   - ラムダ `->` や メソッド参照 `::` のトークン化

3. **問題点と改善案を日本語で報告する**

```
## 字句解析レポート: <ファイル名>

### ファイル概要
- <クラス/ソースの概要>

### トークン化上の注意点
- <トークン種別>: <注意すべき箇所と理由>

### 潜在的な問題
- <問題タイトル>: <説明>
  再現例: `<コード断片>`

### 改善提案
- <提案タイトル>: <内容>
```
