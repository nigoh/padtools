# セキュリティポリシー

## サポート対象バージョン

原則として最新リリースに対してセキュリティ修正を提供します。問題を報告する際は、
利用しているバージョン（`Juml.jar` の `Implementation-Version` または配布物のバージョン）を
明記してください。

## 脆弱性の報告

セキュリティ上の問題を発見した場合は、**公開 Issue では報告しないでください**。
代わりに GitHub の [Security Advisories](https://github.com/nigoh/juml/security/advisories/new)
から非公開でご報告ください。

報告には以下を含めていただけると調査が円滑になります。

- 影響範囲（どの機能・どの入力で発生するか）
- 再現手順と、可能であれば最小再現サンプル
- 想定される影響（情報漏えい・任意コード実行など）
- 利用環境（OS / Java バージョン / Juml のバージョン）

受領後、内容を確認のうえ可能な限り速やかに対応方針を返信します。修正は新しい
リリースとして配布し、報告者のクレジット（希望される場合）を含めて公開します。

## 依存ライブラリの脆弱性

Juml は依存ライブラリを fat jar に同梱して配布します（[`THIRD-PARTY-NOTICES.md`](THIRD-PARTY-NOTICES.md)
参照）。同梱ライブラリに既知 CVE が公開された場合は、依存バージョンを更新したうえで
最新の fat jar を再ビルド（`./gradlew jar`）してご利用ください。
