Claude Code 連携

Juml リポジトリの .claude/ 配下に同梱した、Claude Code 用の エージェント・スキル・スラッシュコマンドのカタログと使い方。AOSP / AAOS の大規模ソース解析と、Juml 自身の Java 解析エンジンの設計相談を支援します。

エージェント 3 スキル 2（+ チートシート 9） コマンド 9 対象 Claude Code
ここで扱うのは Claude Code（Anthropic の CLI / エージェント）向けの定義ファイルであり、Juml 本体（Java/Swing アプリ）の機能ではありません。 Juml 本体の機能は 開発者リファレンス、操作は GUI マニュアル を参照してください。 導入手順の原典はリポジトリの .claude/SETUP_CLAUDE_AGENTS_SKILLS.md です。

3 つの構成要素

Claude Code は .claude/ 配下の定義を読み込み、会話の文脈に応じてエージェント・スキルを自動ロードしたり、ユーザーが入力したスラッシュコマンドを実行したりします。それぞれの役割は次のとおりです。

要素場所役割呼び出し方
エージェント
(agent / subagent)		.claude/agents/*.md特定領域に特化したサブエージェント。独立した文脈で調査・設計を行い、結論だけを親に返す。親エージェントが委譲 / コマンドから起動
スキル
(skill)		.claude/skills/<name>/SKILL.md領域知識・手順書・チートシートのナレッジベース。description に合致する話題で自動ロードされる。文脈一致で自動 / 明示的に参照
スラッシュコマンド
(slash command)		.claude/commands/*.mdユーザーが /name で起動する定型タスク。エージェント委譲やコマンド生成のテンプレート。/コマンド名 引数 を入力
関係性: スラッシュコマンドが入口、エージェントが「考える人」、スキルが「参照する資料」。たとえば /juml-explore aaos --carservice は aaos-juml-explorer エージェントを起動し、そのエージェントが aaos-juml-analyzer スキルの知識を使って解析戦略を設計します。

エージェント（agents）

各エージェントは .md の YAML フロントマター（name / description / tools / model）で定義され、本文に解析方針・入力シグナル・出力形式を記述します。AOSP / AAOS 系は 実行せず読み取り（Read / Grep / Glob）に徹し、Juml の実行は親に委ねる「戦略設計役」です。

エージェント領域ツール役割
aosp-juml-explorer AOSP 汎用 ReadGrepGlob 「○○ を理解したい」という抽象ゴールを受け、どこを・どの順で・どんな仮説で見るか、Juml のどのオプションをなぜ使うかを日本語で提案。Soong (.bp) / AIDL / HAL (.hal) / sepolicy (.te) / partition の解析戦略を設計。
aaos-juml-explorer AAOS（自動車） ReadGrepGlob CarService / VHAL / CarSystemUI / マルチユーザセキュリティなど自動車向けアーキテクチャに特化。CarManager 各種・VHAL・SELinux のブロック図取得戦略を立案。
java-analyst Juml 内部 all tools Juml の Java 解析パイプライン（JavaLexer → JavaStructureExtractor → JavaClassInfo → PlantUML）を熟知した設計相談役。新機能実装・ロジック改善・バグ調査の設計提案を行う。
出力規約: いずれのエージェントも CLAUDE.md に従い「English: 日本語」+「目的:」形式で日本語出力します。AOSP / AAOS 系は提案にとどめ、コマンド実行は親（またはユーザー）に明示的に委ねます。

スキル（skills）

スキルは SKILL.md（前提環境・図化手順・実行例）と複数の cheatsheet-*.md（領域別の深掘り資料）で構成されます。description に書かれたキーワードが会話に現れると自動ロードされ、エージェントや本体スレッドの判断材料になります。

aosp-juml-analyzer

AOSP ソースツリーを Juml で UML 図化する手順集。Soong/Android.bp、AIDL/HAL、partition（system/vendor/product/odm/boot/vbmeta）、SELinux sepolicy (.te) の解析・可視化依頼や、Treble/GKI/VINTF 関連の質問で自動ロードされます。

チートシートカバー範囲
cheatsheet-build.mdSoong / Android.bp / Gradle のビルド構成と依存の読み方
cheatsheet-partition.mdsystem / vendor / product / odm などパーティション分離（Treble）
cheatsheet-hal.mdAIDL / HAL / HIDL インタフェースと VINTF 整合
cheatsheet-sepolicy.mdSELinux ポリシー（.te）・allow/neverallow・avc denied の切り分け

aaos-juml-analyzer

Android Automotive OS（AAOS）ソースツリーを Juml で UML 図化する手順集。CarService/CarManager、Vehicle HAL (VHAL)、CarSystemUI/RRO、MultiUser/権限分離の解析・可視化依頼で自動ロードされます。

チートシートカバー範囲
cheatsheet-carservice.mdCarService / CarManager / ICar AIDL の起動・構造
cheatsheet-vhal.mdVehicle HAL / IVehicle / get・set・subscribe フロー
cheatsheet-ui-rro.mdCarSystemUI / CarLauncher / RRO によるテーマ・レイアウト上書き
cheatsheet-security.mdMultiUser / 権限 / sepolicy によるロール分離（driver / passenger）
前提: スキルは Juml の jar（既定 ~/juml/build/libs/Juml.jar、未ビルドなら ./gradlew jar）と Java 17+ を前提とします。大規模ツリーは -Xmx8g 以上を推奨します。

スラッシュコマンド（commands）

ユーザーが /コマンド名 引数 の形で起動します。$ARGUMENTS を解釈してエージェントに委譲したり、Juml の実行コマンドを生成したりします。.claude/commands/README.md に全体像があります。

AOSP / AAOS 系

コマンド用途例
/juml-explore 統合解析。<layer> <topic> を受け、aosp/aaos 用エージェントとスキルを自動ロードして解析戦略（どこを見るか・どのオプションか・期待される出力）を設計。 /juml-explore aaos --carservice
/juml-explore aosp --build
/juml-explore aosp --method CarService.onCreate
/juml-quick 直接実行。エージェント・スキル不要。図種・パス（・メモリ）を受け、Juml の実行コマンドを素早く生成。class / seq / manifest / deps / summary / all / impact / ref-find / vhal-flow / aidl-binding / er-diagram / data-flow / screen-flow / android-bp / selinux / rro-overlays に対応。 /juml-quick class frameworks/base/services/core
/juml-quick impact android.car.hardware.CarPropertyManager ~/AOSP/...
/juml-quick all ~/AOSP/... 16g
/aosp-help AOSP トピック別クイックリファレンス（build / partition / hal / sepolicy / all）。関連チートシートと解析フローを提示。 /aosp-help sepolicy "avc denied ログの読み方"
/aaos-help AAOS トピック別クイックリファレンス（carservice / vhal / ui / security / all）。 /aaos-help vhal "VHAL の get/set フロー"

Java 解析系（Juml 本体の開発用）

コマンド用途既定の対象
/java-analyzejava-analyst エージェントに解析パイプラインの設計相談・コード分析を依頼。.java パス・機能キーワード・空欄で挙動が変化。パイプライン全体のレビュー
/java-lex指定 Java ファイルを JavaLexer の観点でトークン解析し、字句レベルの問題・エッジケースを調査。JavaLexer.java
/java-structJavaStructureExtractor の抽出結果（クラス/メソッド/フィールド/呼び出し）が正しいか検証。JavaStructureExtractor.java
/java-diagramPlantUML 図（class / sequence / package）の生成ロジックを分析し、出力品質の改善案を提案。PlantUmlClassDiagram

運用系

コマンド用途
/release <version>新バージョンのリリース手順を一括実行。バージョン確認 → 事前チェック → build.gradle 更新 → CHANGE.md のセクション確定 → fat jar ビルド → コミット → タグ push → GitHub リリース → PR 作成、をステップ順に進める（エラー時はユーザーに報告して停止）。
スコープの違い: AOSP/AAOS 系（juml-explore・juml-quick・*-help）は Juml の利用者向け、Java 解析系（java-*）と release は Juml の開発者向けです。

セットアップ

スラッシュコマンドはリポジトリ固有のため、Juml ディレクトリで Claude Code を起動すればそのまま使えます。エージェント・スキルを他プロジェクトでも使いたい場合は、ユーザーホームの ~/.claude/ へコピーします。

# エージェント・スキルをユーザーホームへ配置（任意）
mkdir -p ~/.claude/agents ~/.claude/skills

cp .claude/agents/*.md ~/.claude/agents/
cp -r .claude/skills/aosp-juml-analyzer ~/.claude/skills/
cp -r .claude/skills/aaos-juml-analyzer ~/.claude/skills/

# 確認
ls ~/.claude/agents/ ~/.claude/skills/

AOSP のパスが既定（~/AOSP/）でない場合は環境変数で指定できます。

export AOSP_PATH="/mnt/custom/aosp"

詳細・トラブルシューティングは .claude/SETUP_CLAUDE_AGENTS_SKILLS.md を参照してください。

典型的なワークフロー

  1. 戦略を立てる — /juml-explore aaos --carservice でエージェントに「どこを・どの順で見るか」を設計させる。スキルが自動ロードされ、CarService の構造知識が反映される。
  2. 図を出す — 提案に沿って /juml-quick class <path> や /juml-quick vhal-flow <path> で Juml の実行コマンドを生成・実行し、SVG / Markdown を得る。
  3. 知識を引く — 途中で用語に詰まったら /aosp-help hal / /aaos-help security でチートシートを参照。
  4. （開発者）Juml を改善する — 解析結果が期待と違えば /java-lex / /java-struct / /java-diagram / /java-analyze で原因をパイプライン視点で切り分ける。

独自定義の追加

新しいエージェント・スキル・コマンドを足すときは、既存ファイルの形式に揃えます。

追加先必須フロントマター / 構成方針
agents/*.mdname / description / tools / model + 本文（方針・入力シグナル・出力形式）description はいつ委譲すべきかが分かるよう具体的に。読み取り専用なら tools を絞る。
skills/<name>/SKILL.mdname / description + 前提環境・手順・例。深掘りは cheatsheet-*.md に分割description のキーワードが自動ロードのトリガー。網羅的に列挙する。
commands/*.md（任意）description / allowed-tools + 本文。$ARGUMENTS の解釈を明記ファイル名がコマンド名（foo.md → /foo）。エージェント委譲かコマンド生成かを明確に。
出力は CLAUDE.md のサマリールール（「英語: 日本語」形式と「目的:」の付与）に従ってください。専門用語・固有名詞（Soong, VHAL, PlantUML 等）は原語のまま使います。
