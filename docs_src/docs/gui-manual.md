## 起動と画面構成

### 起動方法

```sh
# GUI を起動
java -jar PadTools.jar

# 起動と同時にプロジェクトを解析開始
java -jar PadTools.jar ~/AndroidStudioProjects/MyApp
```

### 画面レイアウト

```
┌────────────────────────────────────────────────────────────┐
│ メニューバー  (File / Diagram / View / Style / Help)        │
├────────────────────────────────────────────────────────────┤
│ ツールバー上段 (アクション)                                   │
│  Open  Save  Refresh  Back  Search  Scope  Clear Scope     │
│  Zoom+  Zoom-  100%  Fit                                    │
├────────────────────────────────────────────────────────────┤
│ ツールバー下段 (図種選択)                                    │
│  Class  Package  Sequence  Activity  Common                 │
│  Component  Dependency  Manifest  Layout                    │
├─────────────────────┬──────────────────────────────────────┤
│                     │ [Preview][Source][Manifest][Impact]  │
│  プロジェクトツリー   │                                      │
│  (左ペイン)          │     図プレビュー / 各タブ             │
│                     │         (右ペイン)                    │
│                     │                                      │
├─────────────────────┴──────────────────────────────────────┤
│ ステータスバー  (進捗メッセージ / プログレスバー / ズーム率)    │
└────────────────────────────────────────────────────────────┘
```

## プロジェクトを開く

1. **File → Open Project...** (または `Ctrl+O`) を選択
2. ファイルダイアログが開くので、Android / Gradle プロジェクトの**ルートディレクトリ**を選択して「開く」
3. バックグラウンドで解析が開始され、ステータスバーに `Analyzing 1234 / 56789` のような進捗が表示される
4. 解析が完了すると左ペインのツリーにモジュール・パッケージ・クラス・メソッドが展開される

> **ヒント**: 解析途中で気が変わった場合は **File → Cancel Loading** で中断できます。AOSP 級の大規模プロジェクトで有用です。

## ツールバー

### 上段 — アクションボタン

| ボタン | ショートカット | 説明 |
|-------|-------------|------|
| **Open** | `Ctrl+O` | プロジェクトフォルダを選択して開く |
| **Save** | `Ctrl+S` | 現在の図を SVG / PNG / PUML 形式で保存 |
| **Refresh** | `F5` | 現在の図を再生成 |
| **Back** | `Alt+←` | ドリルダウン履歴を 1 段戻す |
| **Search** | `Ctrl+Shift+F` | エンティティ横断検索ダイアログを開く |

## 左ペイン — プロジェクトツリー

プロジェクトの構造を階層ツリーで表示します。

```
▶ [module] app
  ▶ [package] com.example.myapp
    ▶ [class]  MainActivity
      ▷ [method] onCreate
      ▷ [method] onStart
    ▶ [class]  MainViewModel
  ▶ [manifest] AndroidManifest.xml
    ▶ Activities
    ▶ Services
    ▶ Receivers
    ▶ Providers
    ▶ Permissions
    ▶ Features
```

### ツリーの操作

| 操作 | 動作 |
|------|------|
| **クラスノードをクリック** | そのクラスを中心にしたクラス図 (1 ホップ近傍) に切り替える |
| **メソッドノードをクリック** | そのメソッドを起点にしたシーケンス図に切り替える |
| **Manifest 系ノードをクリック** | Manifest 図に自動切替 |
| **パッケージノードを右クリック** | `Show class diagram of this package` でそのパッケージのクラス図にドリルダウン |

> **注意**: パッケージ・クラスは遅延展開 (lazy load) されます。初回クリック時にはやや時間がかかる場合があります。

## 右ペイン — タブ一覧

右ペインには以下の 4 つのタブがあります。

| タブ | 説明 |
|------|------|
| **Preview** | 生成された図を SVG でリアルタイム表示 |
| **PlantUML Source** | 生成された PlantUML テキストをリードオンリーで表示 (デバッグ・コピー用) |
| **Manifest Summary** | AndroidManifest.xml の Markdown サマリー |
| **Impact** | シンボルへの逆参照を層別にたどる Impact 解析 |

## 図種の選択と切り替え

ツールバー下段のボタン、または **Diagram メニュー** のラジオ項目で選択します。

| 図種 | 用途 |
|------|------|
| **Class** | クラス・インターフェースの構造と依存関係 |
| **Package** | パッケージ間の依存関係を高レベルで把握 |
| **Sequence** | メソッド呼び出しの時系列フロー |
| **Activity** | メソッド内の制御フロー (if / for / while など) |
| **Common** | 参照回数が多い「共通クラス」上位 N 件のハイライト |
| **Component** | Android コンポーネント (Activity / Service / Receiver / Provider) の構成 |
| **Dependency** | Gradle モジュール間・外部ライブラリ依存グラフ |
| **Manifest** | AndroidManifest.xml の構造図 |
| **Layout** | Android Layout XML の ViewGroup 階層 |

> **Sequence / Activity / Layout** は起点が未選択のとき、自動的に選択ダイアログを表示します。

## 各図種の詳細操作

### クラス図 (Class)

クラス・インターフェース・列挙体の構造と相互関係を表示します。

**プリセット (密度の切り替え)**

| プリセット | ショートカット | 内容 |
|-----------|-------------|------|
| Minimal | `Ctrl+1` | フィールド非表示・関係線のみ |
| Balanced | `Ctrl+2` | デフォルト (フィールド + 主要メソッド) |
| Detailed | `Ctrl+3` | フィールド + 全メソッド + アノテーション |

**ドリルダウン**

- **プレビュー上のクラス名をクリック** → そのクラスを中心に 1 ホップ近傍の詳細図に切り替え
- **プレビュー上のクラス名を右クリック** → メソッド一覧ポップアップが出て、選択するとシーケンス図に切り替え

---

### パッケージ図 (Package)

パッケージごとのクラス数と参照関係 (継承 / 実装 / フィールド型) を集約して表示します。  
**Scope ダイアログ**でパッケージや正規表現フィルタを指定すると表示対象を絞り込めます。

---

### シーケンス図 (Sequence)

メソッドの呼び出し連鎖を時系列で図示します。

**起点の選択**

1. **Diagram → Choose Sequence Entry...** を選択 (または Sequence ボタンをクリックして自動表示)
2. SequenceEntryDialog でメソッド名を入力して候補を絞り込む (例: `MainActivity.onCreate`)
3. 候補一覧から選択して **OK**

**登場人物 (Participant) のフィルタ**

| 操作 | 説明 |
|------|------|
| Diagram → Filter Sequence Participants... | 非表示にしたい Participant のチェックを外す |
| Diagram → Clear Sequence Participant Filter | フィルタを全解除 |

**再帰トレース深さ**

デフォルトは 5 層。深すぎる場合は Style Settings からコメント表示を抑制したり、Participant フィルタで外部ライブラリを隠すと図が見やすくなります。

---

### アクティビティ図 (Activity)

メソッド内の制御フロー (`if` / `while` / `switch` / `try` など) を図示します。

1. **Diagram → Choose Activity Method...** を選択
2. 起点メソッドを選択して **OK**

---

### 共通クラス図 (Common)

他クラスから参照される回数 (fan-in) が多い「共通クラス」の上位 N 件 (既定 20 件) を強調表示します。  
設計上のボトルネックや過度な依存候補を素早く発見するのに役立ちます。

---

### コンポーネント図 (Component)

AndroidManifest.xml から読み取った Activity / Service / BroadcastReceiver / ContentProvider の構成と、`exported` 属性・ランチャー設定・uses-permission を一覧表示します。

---

### Gradle 依存グラフ (Dependency)

モジュール間の `project(':x')` 依存と外部 Maven ライブラリを可視化します。  
`libs.versions.toml` (Version Catalog) も自動解決します。

---

### Manifest 図 (Manifest)

AndroidManifest.xml の `<application>` 属性を中央ノードに配置し、配下のコンポーネントと `uses-permission` / `uses-feature` を図示します。

---

### レイアウト図 (Layout)

1. **Diagram → Choose Layout File...** を選択 (または Layout ボタンをクリックして自動表示)
2. Android Layout XML ファイルを選択
3. `ViewGroup` 階層が図として出力される

---

## スコープ (表示範囲) の絞り込み

大規模プロジェクトで表示クラスが多すぎる場合、スコープを絞って見やすい図を生成できます。

### ダイアログを開く

- ツールバーの **Scope** ボタン、または **Diagram → Scope...**

### 設定項目

| 項目 | 説明 |
|------|------|
| **Include packages** | 表示するパッケージを選択 (複数可) |
| **Exclude packages** | 除外するパッケージを選択 (複数可) |
| **Modules** | 対象モジュールを選択 (複数可) |
| **Class name regex** | クラス名の正規表現フィルタ |
| **Max classes** | 表示する最大クラス数 |
| **Seed class + neighbor hops** | 特定クラスを中心に N ホップ以内のクラスのみ表示 |
| **Relation kinds** | 表示する関係線の種類 (継承 / 実装 / 使用) |
| **Visibility** | 可視性フィルタ (PUBLIC_ONLY など) |
| **Exclude external libraries** | 外部ライブラリのクラスを除外 |
| **Parse mode** | Full (全解析) / Headers-only (高速・軽量) |
| **Preset** | Minimal / Balanced / Detailed を一括適用 |

### スコープのリセット

- ツールバーの **Clear Scope** ボタン、または **Diagram → Clear Scope**

## エンティティ検索

クラス・メソッド・フィールドをプロジェクト横断で検索できます。

1. **Diagram → Search Entities...** (または `Ctrl+Shift+F`)
2. テキストボックスに名前の一部を入力 → リアルタイムで候補ツリーが絞り込まれる
3. **Kind チェックボックス**で対象を絞る: CLASS / METHOD / FIELD
4. 候補を選択して操作:

| ボタン | 動作 |
|--------|------|
| **OK** | CLASS → そのクラスのクラス図<br>METHOD → そのメソッドのシーケンス図<br>FIELD → そのフィールドが属するクラスのクラス図 |
| **Drill-down** | DETAILED プリセットで詳細なクラス図を表示 |

## ズーム・パン操作

### キーボード / メニュー

| 操作 | ショートカット |
|------|-------------|
| ズームイン | `Ctrl+=` |
| ズームアウト | `Ctrl+-` |
| ズームリセット (100%) | `Ctrl+0` |
| 図全体をウィンドウに合わせる | `Ctrl+F` |

### マウス

| 操作 | 動作 |
|------|------|
| `Ctrl+ホイール` | ズームイン / アウト |
| **左ドラッグ** | 図をパン (移動) |

ズーム倍率はステータスバー右端に常時表示されます (例: `150%`)。  
倍率の範囲は **0.1 倍 〜 8.0 倍** です。

## ドリルダウンと履歴ナビゲーション

### ドリルダウン (詳細へ)

- **プレビュー上でクラス名の上にカーソルを置く** → カーソルが指差し (手のひらアイコン) に変わったらクリック対象
- **左クリック** → そのクラスを seed に DETAILED プリセットで詳細図へ移動
- **右クリック** → メソッド一覧ポップアップ → メソッドを選んでシーケンス図へ

### 履歴を戻る / 進む

| 操作 | ショートカット |
|------|-------------|
| 1 段前の図に戻る | `View → Back` または `Ctrl+Alt+←` |
| 1 段先の図に進む | `View → Forward` または `Ctrl+Alt+→` |

ツールバーの **Back** ボタン (`Alt+←`) でも 1 段戻れます。

| テーマ名 | 印象 |
|---------|------|
| (None) | PlantUML デフォルト |
| plain | シンプルな白黒 |
| cerulean | 青系のクリーン |
| sketchy | 手書き風 |
| mono | モノクロ |
| vibrant | 鮮やかな配色 |
| materia | マテリアルデザイン風 |
| hacker | 黒背景・緑文字 |
| cyborg | ダークサイバー風 |
| mars | 赤系 |
| amiga | レトロ風 |
| spacelab | 宇宙研究所風 |

### 詳細設定 (Style Settings...)

**Style → Style Settings...** でダイアログを開きます。

**共通設定**

| 項目 | 説明 |
|------|------|
| テーマ | 上記テーマを選択 |
| 背景色 | カスタム背景色を指定 |

| 項目 | 説明 |
|------|------|

| 項目 | 説明 |

## 図の保存・エクスポート

1. **File → Save Diagram As...** (または `Ctrl+S`)
2. ファイルダイアログで出力先・ファイル名を入力
3. 拡張子で形式を自動判別:

| 拡張子 | 出力形式 |
|--------|---------|
| `.svg` | SVG ベクター (高品質・無制限サイズ) |
| `.png` | PNG ラスター (4096×4096 上限あり) |
| `.puml` / `.txt` | PlantUML テキスト (そのまま保存) |

> **ヒント**: 大きな図は `.svg` での出力を推奨します。PNG は PlantUML のキャンバスサイズ上限 (既定 4096×4096) に達すると切り詰められます。

## Impact 解析

あるクラスやメソッドに**依存している箇所 (逆参照)** を BFS で辿り、変更影響範囲を把握する機能です。

1. 右ペインの **Impact** タブをクリック
2. **Symbol** 欄に対象の完全修飾名を入力
   - クラス: `com.example.myapp.MyClass`
   - メソッド: `com.example.myapp.MyClass.myMethod`
3. **Depth** スピナーで BFS 深さを指定 (既定: 3)
4. **Analyze** ボタンを押す
5. バックグラウンド処理後、層別の呼び出し元ツリーが表示される  
   各ノードには**変更影響リスクスコア**が付与されます

> **注意**: 初回の Analyze 実行時は参照インデックスの構築に時間がかかります。2 回目以降はキャッシュが利用されます。
